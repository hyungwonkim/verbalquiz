**Summary**
I want to make a verbal quiz generator that generates a requested number of synonym questions and analogy questions and antonym questions based on the words by grades. The list of words are from data/verbal_word_bank.csv, which has word, grade, and category. It should work in the web-app, where user will select the grade level, and the number of questions to generate per question type; synonym, antonym, and analogy. The question generator relies on the static question bank organized by grade, word, and analogy. 

**Dataset**
Static complete vocabulary set: ddata/verbal_word_bank.csv, which has word, grade, and category. All questions and choices come from the words in this list. 

**Example Problem**
Synonym:
BENEVOLENT:
(A) Generous  (B) Cruel  (C) Timid  (D) Wise

Analogy:
PUPPY : DOG ::
(A) Calf : Cow  (B) Feather : Bird  (C) Cat : Kitten  (D) Horse : Stable

Antonym: 
Choose the word that is the exact opposite of ENORMOUS: 
(A) Soft   (B) Average  (C) Tiny  (D) Weak

**Project Execution Steps**
1. First create a static mapping from each word and grade to a list of synonyms and antonyms. For each grade and each word, make a list of synonyms and antonyms by scanning the pool of words *in the same grade or lower*. They must match the category of the word, such that a noun's synonym and antonym must be all nouns. For example, for grade 2's word "calm", search for all synonym and antonym of the adjective "calm" in the list of grade 2 and grade 1 words. If the word does not have a synonym or antonym in the list of words for the grade and lower grades, leave it empty. But prioritize the recall over precision in this stage. 

2. Use an independent agent to verify the mapping generated by the step 1. If a word marked as synonym or antonym is too loosely related, churn it. This stage's task is to priortize precision over recall since the trimmed list coming from this stage will be served as a the pool of synonym and antonym questions. Synonym and antonym question generators will simply pull words from this organized list by grade and words.

3. Make a question generator for synonym and antonym. The synonym generator takes two arguments, a given number N of questions and grade G. It randomly selects N words from the mapping created in the previous step of the grade G, where the synonym list is not empty, and select one synonym randomly. For each question, select 3 random distractors of the same vocab category from the same grade and make sure that they do not belong to the synonym of the chosen question word. Make the antonym generator using the similar logic.

4. Pre-populate analogy questions per grade. Per grade, create analogy questions similar to the example:
```
Analogy:
PUPPY : DOG ::
(A) Calf : Cow  (B) Feather : Bird  (C) Cat : Kitten  (D) Horse : Stable
```
For each grade, you can use any words from the grade or lower grades. For different grades, questions can repeat but within the grade, all questions must be unique. Generate at least the following number of analogy questions per grade:
Grade 1: 50 questions
Grade 2: 100 questions
Grade 3: 150 questions
Grade 4: 200 questions
Grade 5: 250 questions
Grade 6: 300 questions
These numbers are the minimum requirement. You should put best effort to make as many as possible. Organize the analogy questions by grade. 

5. Critique for the analogy questions generated in the previous stage. Try solving the analogy questions and make sure they are sensible. If you find a non-sensible or too niche analogy question, remove it so that the final list is refined. Make sure that the last refined version has at least the number of questions per grade of the mininum requirements speficied in the previous section. This list will be the source of analogy questions generator. 

6. Make an analogy question generator. It takes two arguments, the number of questions to generate, N, and the grade G. Pick N random questions without repeatition (unless N is not greater than the number of questions in the pool, otherwise return all questions for the grade) from the grade G of the pool generated in the previous step.
